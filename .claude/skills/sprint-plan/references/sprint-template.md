# Sprint Template

Use this template when writing draft sprints in Phase 3 and the final merged
sprint in Phase 6. Adapt section depth to the scope of the work.

---

```markdown
# Sprint NNN: [Title]

## Overview

2-3 paragraphs on the "why" and high-level approach.

## Use Cases

1. **Use case name**: Description of the scenario this enables
2. ...

## Architecture

Component descriptions, data flow, and diagrams (ASCII art or Mermaid).

## Implementation

### Phase 1: [Name] (~X% of effort)

**Files:**
- `path/to/file` — Description of changes

**Tasks:**
- [ ] Task 1
- [ ] Task 2

### Phase 2: [Name] (~X% of effort)

...

## Files Summary

| File | Action | Purpose |
|------|--------|---------|
| `path/to/file` | Create / Modify / Delete | Description |

## Definition of Done

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Tests pass
- [ ] No regressions

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| ... | Low/Med/High | Low/Med/High | ... |

## Security Considerations

- Item 1 (omit this section if not applicable)

## Dependencies

- Prior work (if any)
- External requirements

## Open Questions

Uncertainties needing resolution before or during implementation.
```
